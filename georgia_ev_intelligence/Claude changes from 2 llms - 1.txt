     A. Best Retrieval

     A1. Enable hybrid (dense + sparse) — highest ROI, collection already 
     supports it

     The Qdrant collection is configured with both dense (768-dim cosine) and
     sparse (BM25 with IDF)
     named vectors. The sparse vector is currently bypassed in
     retrieve_and_rerank(). Enabling it
     requires only a small change to _search_dense().

     Why this matters: Dense embeddings miss exact terms: company names spelled
     differently,
     "DC-to-DC", "powder coating", "Tier 1", county names, OEM names like
     "Hyundai" or "Kia". BM25
     catches these by exact token match. Hybrid = best of both.

     How to implement (in retrieval.py):

     from qdrant_client import models

     def _search_hybrid(client, collection, dense_name, sparse_name,
                        query_vector, sparse_vector, top_k):
         results = client.query_points(
             collection_name=collection,
             prefetch=[
                 models.Prefetch(query=query_vector, using=dense_name,
     limit=top_k),
                 models.Prefetch(query=sparse_vector, using=sparse_name,
     limit=top_k),
             ],
             query=models.FusionQuery(fusion=models.Fusion.RRF),
             limit=top_k,
             with_payload=True,
         )
         # ... build hits list same as _search_dense

     Build the sparse vector from the query text using the same hash-based
     term-frequency logic in
     phase2_embedding/vector_store.py. If the collection has no sparse vector
